Failure-lab endpoints

Endpoint 운영 의미 예상 관찰 지점
/api/failure-lab/was-runtime 내장 Tomcat 기반 WAS process 설정과 JVM thread 상태 확인 Tomcat maxThreads, acceptCount, request thread name, live JVM threads
/api/failure-lab/sleep?millis=3000 WAS request thread를 일정 시간 점유하는 긴 요청 Nginx upstream response time, app request duration, 동시 요청 시 WAS thread saturation
/api/failure-lab/db-sleep?millis=3000 PostgreSQL pg_sleep 기반 DB dependency 지연 DB dependency latency, readiness와 DB-backed endpoint 차이, connection wait 관찰
/api/failure-lab/db-hold?seconds=10 HikariCP connection을 일정 시간 점유하는 connection pressure workload HikariCP active/idle/awaiting 상태, 업무 API 지연, PostgreSQL pg_stat_activity
/api/failure-lab/db-pool 현재 HikariCP 설정과 pool 상태 확인 maximumPoolSize, connectionTimeout, active/idle/awaiting connection evidence
/api/failure-lab/file-storage-check NFS/app file storage mount 상태 확인 파일 저장소 장애, 권한 문제, 업로드 실패 원인 분리
/api/failure-lab/upload-limits WAS multipart upload 제한 확인 Nginx client_max_body_size와 WAS multipart limit 비교
